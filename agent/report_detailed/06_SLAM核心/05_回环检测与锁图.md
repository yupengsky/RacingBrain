# 回环检测与锁图

## 1. 模块位置

回环检测器定义：

```text
slam/slam/include/slam/loop_closure_detector.hpp
```

实现：

```text
slam/slam/src/loop_closure_detector.cpp
```

类名：

```cpp
EnhancedLoopClosureDetector
```

## 2. 当前回环检测的定位

当前实现不是传统 SLAM 中的视觉词袋、Scan Context 或图优化回环。

它是一个基于 GNSS/INS 轨迹的几何回环检测器：

```text
如果车辆经过足够时间后再次接近起点，
并且运动方向朝向起点，
并且接近速度足够，
并且连续多帧满足，
则认为完成一圈。
```

它主要服务于循迹赛道模式：

```text
第一圈探索建图 -> 回到起点 -> 锁图 -> 后续圈使用稳定地图
```

## 3. 参数

构造函数和 `configure` 支持参数：

```cpp
double distance_threshold
double approach_angle_threshold
int min_consecutive_detections
int max_consecutive_detections
double min_time_between_detections
int history_buffer_size
double min_approach_speed
double min_lap_time
```

YAML 中 `autocross.yaml` 配置：

```yaml
loop_closure:
  distance_threshold: 1.5
  approach_angle_threshold: 35.0
  min_approach_speed: 0.8
  min_consecutive_detections: 2
  max_consecutive_detections: 5
  min_time_between_detections: 3.0
  history_buffer_size: 10
  min_lap_time: 20.0
```

含义：

- `distance_threshold`：距离起点多近算接近。
- `approach_angle_threshold`：运动方向与指向起点方向夹角阈值。
- `min_approach_speed`：向起点接近的速度投影下限。
- `min_consecutive_detections`：冲刺圈触发所需连续帧数。
- `max_consecutive_detections`：探索圈触发所需连续帧数。
- `min_time_between_detections`：回环触发冷却时间。
- `history_buffer_size`：用于估计运动方向和速度的历史长度。
- `min_lap_time`：最小圈时，避免刚起步误触发。

## 4. 起点设置

SLAM 同步回调中第一次初始化：

```cpp
if (!is_lcd_initialized_) {
    loop_detector_.setOrigin(Eigen::Vector3d(0.0, 0.0, 0.0));
    is_lcd_initialized_ = true;
}
```

起点被设置为局部地图原点：

```text
(0, 0, 0)
```

因为第一帧 GNSS 已经作为 SLAM 局部地图原点。

`setOrigin` 内部：

```cpp
origin_position_ = position;
is_loop_origin_set_ = true;
is_start_time_set_ = false;
consecutive_detections_ = 0;
loop_closed_ = false;
position_history_.clear();
timestamp_history_.clear();
```

## 5. detectLoopClosure 主流程

函数：

```cpp
bool detectLoopClosure(
    double current_time,
    const Eigen::Vector3d& current_position,
    bool is_exploration_lap
)
```

输入：

- 当前时间。
- 当前车辆位置。
- 是否探索圈。

输出：

- `true`：检测到回环。
- `false`：没有回环。

## 6. 第一步：检查起点是否设置

```cpp
if (!is_loop_origin_set_) return false;
```

没有起点就无法回环。

## 7. 第二步：记录起始时间

首次调用时：

```cpp
if (!is_start_time_set_) {
    start_time_ = current_time;
    is_start_time_set_ = true;
}
```

后续根据 `current_time - start_time_` 判断最小圈时。

## 8. 第三步：最小圈时检查

```cpp
double elapsed_time = current_time - start_time_;
if (elapsed_time < min_lap_time_) {
    return false;
}
```

目的：

- 防止刚起步时车辆还在原点附近，被误判回环。

默认：

```text
20 seconds
```

## 9. 第四步：更新历史轨迹

```cpp
updateHistory(current_time, current_position);
```

内部：

```cpp
position_history_.push_back(position);
timestamp_history_.push_back(timestamp);
if (position_history_.size() > history_buffer_size_) {
    position_history_.pop_front();
    timestamp_history_.pop_front();
}
```

历史轨迹用于：

- 判断运动方向。
- 计算接近速度。

## 10. 第五步：冷却时间

```cpp
if (loop_closed_ && (current_time - last_detection_time_ > min_time_between_detections_)) {
    loop_closed_ = false;
    consecutive_detections_ = 0;
}
```

回环触发后短时间内不允许重复触发。

## 11. 第六步：连续检测阈值

```cpp
int detection_threshold = is_exploration_lap
    ? max_consecutive_detections_
    : min_consecutive_detections_;
```

设计：

- 探索圈速度较慢，可以要求更多连续帧，避免误触发。
- 冲刺圈速度快，要求较少连续帧，否则可能来不及触发。

注意命名：

- `max_consecutive_detections_` 用于探索圈。
- `min_consecutive_detections_` 用于冲刺圈。

## 12. 条件一：距离起点足够近

```cpp
double distance_to_origin = calculateDistanceToOrigin(current_position);
bool distance_condition = (distance_to_origin < distance_threshold_);
```

距离：

```cpp
return (position - origin_position_).norm();
```

默认阈值：

```text
1.5 m
```

## 13. 条件二：车辆朝向起点运动

函数：

```cpp
bool isMovingTowardsOrigin(const Eigen::Vector3d& current_position) const
```

需要至少两帧历史：

```cpp
if (position_history_.size() < 2) return false;
```

运动方向：

```cpp
Eigen::Vector3d current_to_prev =
    position_history_.back() - position_history_[position_history_.size()-2];
```

指向起点方向：

```cpp
Eigen::Vector3d current_to_origin = origin_position_ - current_position;
```

夹角：

```cpp
double cos_angle = motion_direction.dot(to_origin_direction);
return (std::acos(cos_angle) < approach_angle_threshold_);
```

默认阈值：

```text
35 degrees
```

## 14. 条件三：接近速度足够

函数：

```cpp
bool hasSufficientApproachSpeed() const
```

速度：

```cpp
Eigen::Vector3d diff = position_history_.back() - position_history_[position_history_.size()-2];
double dt = timestamp_history_.back() - timestamp_history_[timestamp_history_.size()-2];
Eigen::Vector3d vel = diff / dt;
```

指向起点方向：

```cpp
Eigen::Vector3d to_origin = (origin_position_ - position_history_.back()).normalized();
```

速度投影：

```cpp
return (std::abs(vel.dot(to_origin)) >= min_approach_speed_);
```

默认阈值：

```text
0.8 m/s
```

这里使用了绝对值，因此严格说是“沿起点方向轴线的速度分量足够”，不区分正负。方向条件已经在上一关检查。

## 15. 连续满足计数

```cpp
if (distance_condition && direction_condition && speed_condition) {
    consecutive_detections_++;
} else {
    consecutive_detections_ = 0;
}
```

只有连续满足所有条件，计数才增长。一旦中断就归零。

## 16. 回环触发

```cpp
if (consecutive_detections_ >= detection_threshold &&
    !loop_closed_ &&
    (current_time - last_detection_time_ > min_time_between_detections_)) {

    loop_detected = true;
    loop_closed_ = true;
    last_detection_time_ = current_time;
}
```

返回：

```cpp
return loop_detected;
```

## 17. SLAM 中回环后的动作

`processAutocrossTrack` 中：

```cpp
if (loop_closed) {
    lap_count_++;

    if (!map_locked_) {
        // 第一圈完成
        erase unstable cones;
        map_locked_ = true;
    } else {
        // 后续圈完成
    }

    loop_detector_.resetLoopStatus();
}
```

## 18. resetLoopStatus

```cpp
void EnhancedLoopClosureDetector::resetLoopStatus() {
    loop_closed_ = false;
    consecutive_detections_ = 0;
}
```

用于回环处理结束后重新进入检测状态。

## 19. 回环检测复现伪代码

```cpp
bool detect_loop(time, pos, exploration_lap) {
    if !origin_set:
        return false;

    if !start_time_set:
        start_time = time;
        start_time_set = true;

    if time - start_time < min_lap_time:
        return false;

    update_history(time, pos);

    if loop_closed and time - last_detection_time > cooldown:
        loop_closed = false;
        consecutive = 0;

    threshold = exploration_lap ? max_consecutive : min_consecutive;

    dist_ok = norm(pos - origin) < distance_threshold;
    direction_ok = angle(motion_direction, origin - pos) < angle_threshold;
    speed_ok = abs(velocity dot normalize(origin - pos)) > min_speed;

    if dist_ok and direction_ok and speed_ok:
        consecutive += 1;
    else:
        consecutive = 0;

    if consecutive >= threshold and !loop_closed and time - last_detection_time > cooldown:
        loop_closed = true;
        last_detection_time = time;
        return true;

    return false;
}
```

## 20. 工程意义

回环检测本身不修正地图坐标，也不做图优化。它的工程意义是“切换建图阶段”：

```text
未回环：
  可以新增锥桶，允许地图生长。

第一圈回环：
  删除不稳定锥桶，锁定地图。

锁图后：
  不再新增锥桶，减少高速阶段误检污染。
```

这对赛车任务非常实用，因为比赛通常有探索圈和冲刺圈的节奏。
