# 地图维护、发布与赛道模式

## 1. 本章目标

上一章解释了每个观测如何进入地图。本章解释：

- 每一帧结束后如何惩罚漏检锥桶。
- 如何删除不可信锥桶。
- 如何判定稳定锥桶。
- 如何发布全局地图。
- 三种赛道模式差异。

## 2. 视野判断

函数：

```cpp
bool isInFieldOfView(const Eigen::Vector2d& global_pos,
                     const Eigen::Matrix4d& current_pose)
```

输入：

- `global_pos`：地图中已有锥桶全局坐标。
- `current_pose`：当前车辆在地图中的位姿。

流程：

### 2.1 全局点转车辆局部坐标

```cpp
Eigen::Vector4d p_global(global_pos.x(), global_pos.y(), 0.0, 1.0);
Eigen::Vector4d p_local = current_pose.inverse() * p_global;
```

### 2.2 提取局部坐标

```cpp
double x = p_local.x();  // 车头方向
double y = p_local.y();  // 车左方向
```

### 2.3 计算距离和角度

```cpp
double dist = std::sqrt(x*x + y*y);
double angle = std::atan2(std::abs(y), x);
```

### 2.4 判断视野

```cpp
return (
    x > 0 &&
    dist > blind_range &&
    dist < max_range &&
    angle < half_fov
);
```

参数：

- `lidar_blind_range`
- `max_lidar_range`
- `fov_angle`

含义：

```text
锥桶必须在车前方，在最大感知范围内，不在近距离盲区，且位于 FOV 扇形角内。
```

## 3. miss 惩罚

帧末遍历地图：

```cpp
for (auto it = cone_map_.begin(); it != cone_map_.end(); ) {
    bool in_view = isInFieldOfView(it->pos, historical_T_veh_to_map);
    if (in_view && !it->matched_this_frame) {
        it->existence_score += params_.l_miss;
    }
    ...
}
```

设计思想：

- 如果锥桶不在视野里，没看到是正常的，不扣分。
- 如果锥桶在视野里，但本帧没被任何观测匹配，说明它可能是误检或已经不存在，扣分。

`l_miss` 是负数。

## 4. 删除不可信锥桶

直线模式：

```cpp
if (it->existence_score <= params_.l_min) {
    it = cone_map_.erase(it);
}
```

循迹模式：

```cpp
if (it->existence_score < params_.l_min) {
    it = cone_map_.erase(it);
}
```

差别只是 `<=` 和 `<`，实质影响很小。

## 5. 稳定锥桶判定

如果没有删除：

```cpp
it->is_stable = (it->existence_score > params_.l_stable);
```

只有 `is_stable=true` 的锥桶会被发布到 `/global_map`。

这就是分级发布策略的代码实现：内部会跟踪更多候选锥桶，但只对外发布稳定锥桶。

## 6. 直线模式 processAccelerationTrack

函数：

```cpp
processAccelerationTrack
```

流程：

1. 遍历融合地图中的所有锥桶。
2. 跳过 `UNKNOWN`。
3. 局部坐标转全局坐标。
4. 解析颜色。
5. 计算距离和旋转矩阵。
6. 调用 `updateMap`。
7. 帧末对视野内未匹配锥桶扣分。
8. 删除低分锥桶。
9. 更新稳定状态。

直线模式没有回环，没有锁图。

适用：

```text
Acceleration 直线加速赛道。
```

## 7. 循迹模式 processAutocrossTrack

函数：

```cpp
processAutocrossTrack
```

前半部分与直线模式相同：

- 遍历观测。
- 跳过 UNKNOWN。
- 坐标变换。
- 调用 `updateMap`。

区别在于：

### 7.1 锁图后不做锥桶删减

```cpp
if (!map_locked_) {
    // 地图维护和删除
}
```

如果已锁图，不再执行 miss 惩罚和删除。

设计意图：

- 第一圈探索时建立稳定地图。
- 锁图后地图固定，避免高速冲刺时因为漏检导致锥桶被删。

### 7.2 执行回环检测

```cpp
bool loop_closed = loop_detector_.detectLoopClosure(
    current_time,
    Eigen::Vector3d(tx, ty, tz),
    !map_locked_
);
```

如果检测到回环：

- `lap_count_++`
- 如果还没锁图，执行第一圈完成逻辑。
- 如果已经锁图，记录后续圈完成。
- 重置回环检测器状态。

## 8. 第一圈回环后的锁图

代码：

```cpp
if (!map_locked_) {
    RCLCPP_INFO(..., "=== LAP 1 COMPLETED! Map Locked! ===");

    int all_cones = cone_map_.size();
    cone_map_.erase(
        std::remove_if(cone_map_.begin(), cone_map_.end(),
                       [](const GlobalCone& c) { return !c.is_stable; }),
        cone_map_.end()
    );

    map_locked_ = true;
}
```

含义：

- 第一圈结束时，删除所有不稳定锥桶。
- 只保留稳定地图。
- 设置 `map_locked_=true`。
- 后续 `updateMap` 不再新增锥桶。

这是循迹模式的核心策略。

## 9. 后续圈

如果已经锁图后再次回环：

```cpp
RCLCPP_INFO(this->get_logger(), "LAP %d COMPLETE. ...", lap_count_);
```

工程意图：

```text
第一圈探索建图，后续圈快速通过。
```

## 11. 阿克曼 ROI

函数：

```cpp
bool isInAckermannTube(double x, double y)
```

用于控制新锥桶是否允许加入地图。

### 11.1 基础长度过滤

```cpp
if (x > max_tube_length_ || x < 0.0) {
    return false;
}
```

锥桶必须在车前方且不超过最大管道长度。

### 11.2 直线时退化为矩形 ROI

```cpp
if (std::abs(current_gyro_z_) < params_.max_straight_angular) {
    return std::abs(y) < half_tube_width_;
}
```

也就是说：

```text
0 < x < max_tube_length
|y| < half_tube_width
```

### 11.3 转弯时使用圆弧管道

```cpp
double v = current_speed_;
double R = v / current_gyro_z_;
double dist_to_center_of_circle = std::sqrt(x*x + (y-R)*(y-R));
double lateral_deviation = std::abs(dist_to_center_of_circle - std::abs(R));
return lateral_deviation < half_tube_width_;
```

含义：

- 阿克曼车辆转弯近似沿圆弧运动。
- 圆心在车辆侧向 `R` 处。
- 锥桶到圆弧轨迹的横向偏差小于管道半宽才接受。

## 12. 地图发布 publishGlobalMap

函数：

```cpp
void publishGlobalMap(const builtin_interfaces::msg::Time& stamp)
```

发布类型：

```text
visualization_msgs/msg/MarkerArray
```

### 12.1 先删除上一帧

```cpp
visualization_msgs::msg::Marker delete_all;
delete_all.header.frame_id = sys_.map_frame;
delete_all.header.stamp = stamp;
delete_all.action = 3; // DELETEALL
map_msg.markers.push_back(delete_all);
```

### 12.2 只发布稳定锥桶

```cpp
for (const auto &cone : cone_map_) {
    if (!cone.is_stable)
        continue;
    ...
}
```

### 12.3 使用 MeshResource

```cpp
m.type = visualization_msgs::msg::Marker::MESH_RESOURCE;
m.mesh_use_embedded_materials = false;
```

大黄锥桶：

```cpp
m.mesh_resource = "package://slam/meshes/cone_big.dae";
```

其他锥桶：

```cpp
m.mesh_resource = "package://slam/meshes/cone.dae";
```

### 12.4 位置与颜色

```cpp
m.pose.position.x = cone.pos.x();
m.pose.position.y = cone.pos.y();
m.pose.position.z = 0.0;
m.color.r = cone.r;
m.color.g = cone.g;
m.color.b = cone.b;
m.color.a = 1.0;
```

## 13. 发布日志

每秒节流输出：

```cpp
RCLCPP_INFO_THROTTLE(
    this->get_logger(),
    *this->get_clock(),
    1000,
    "Map Status: Total Tracked: %ld | Confirmed Stable: %d",
    cone_map_.size(),
    published_count
);
```

其中：

- `Total Tracked`：内部跟踪的所有锥桶数量。
- `Confirmed Stable`：发布出去的稳定锥桶数量。

## 14. 地图维护复现伪代码

```cpp
void finish_frame_update(T_vehicle_to_map) {
    for cone in cone_map:
        in_view = is_in_fov(cone.pos, T_vehicle_to_map);

        if in_view and !cone.matched_this_frame:
            cone.score += l_miss;

        if cone.score <= l_min:
            erase(cone);
        else:
            cone.stable = cone.score > l_stable;
}

void publish_global_map() {
    markers.push_back(delete_all_marker);

    for cone in cone_map:
        if !cone.stable:
            continue;

        marker = make_cone_mesh_marker(cone.pos, cone.color);
        markers.push_back(marker);

    pub.publish(markers);
}
```

## 15. 汇报时怎么讲这一章

可以总结为：

```text
系统不是看到锥桶就立刻发布，而是维护一个带生命值和协方差的全局锥桶仓库。
锥桶多次被看到会加分并逐渐稳定；如果本该被看到却漏检，会扣分并最终删除。
只有稳定锥桶才发布给 RViz，这样减少地图闪烁和噪声污染。
```
