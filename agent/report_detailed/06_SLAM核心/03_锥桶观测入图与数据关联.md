# 锥桶观测入图与数据关联

本章是整个工程最核心的技术部分。只要复现本章内容，就能复现 SLAM 后端的核心功能。

## 1. 输入和目标

输入：

```text
同步后的 GNSS/INS 位姿
融合后的 drd25_msgs/Map
```

融合地图中的每个锥桶：

```text
cone_obs.x
cone_obs.y
cone_obs.color
```

目标：

```text
把当前帧局部锥桶观测融合进全局 cone_map_
```

全局地图容器：

```cpp
std::vector<GlobalCone> cone_map_;
```

## 2. 每帧建图入口

函数：

```cpp
syncCallback
```

进入建图前：

```cpp
std::lock_guard<std::mutex> lock(map_mutex_);

for (auto &cone : cone_map_) {
    cone.matched_this_frame = false;
}
```

作用：

- 加锁，防止地图并发访问。
- 每一帧开始时，把所有已知锥桶标记为“本帧未匹配”。

后续如果某个锥桶被当前观测匹配，就设为 `true`。帧末会对视野内但没匹配的锥桶扣分。

## 3. 赛道策略分发

```cpp
switch (current_track_type_) {
    case TrackType::ACCELERATION:
        processAccelerationTrack(map_msg, historical_T_veh_to_map);
        break;
    case TrackType::AUTOCROSS:
        processAutocrossTrack(map_msg, historical_T_veh_to_map, ...);
        break;
    case TrackType::SKIDPAD:
        processSkidpadTrack(map_msg, historical_T_veh_to_map, ...);
        break;
}
```

直线和循迹在“观测入图”部分基本一致，区别主要在地图维护和回环锁图。

## 4. 跳过 UNKNOWN

对每个融合锥桶：

```cpp
if (cone_obs.color == drd25_msgs::msg::Cone::UNKNOWN)
    continue;
```

这意味着：

- 融合层输出 Unknown 锥桶不会进入 SLAM 地图。
- 如果相机颜色没有识别成功，这个 LiDAR 位置不会被用于建图。

这个策略提高了语义地图纯度，但可能牺牲召回。

## 5. 坐标转换

每个观测先从 LiDAR 局部坐标转全局：

```cpp
Eigen::Vector4d p_lidar(cone_obs.x, cone_obs.y, 0.0, 1.0);
Eigen::Vector4d p_global = historical_T_veh_to_map * T_l2v_ * p_lidar;
```

然后：

```cpp
double gx = p_global.x();
double gy = p_global.y();
```

这些全局坐标进入 `updateMap`。

## 6. 颜色解析

函数：

```cpp
parseConeColor(cone_obs.color, r, g, b);
```

映射：

```text
BLUE          -> r=0, g=0, b=1
RED           -> r=1, g=0, b=0
YELLOW_BIG    -> r=1, g=1, b=0
YELLOW_SMALL  -> r=1, g=1, b=0
UNKNOWN       -> r=0.5, g=0.5, b=0.5
```

`YELLOW_BIG` 和 `YELLOW_SMALL` 可视化颜色相同，但 type 不同。

## 7. 观测距离

局部距离：

```cpp
double dist = std::sqrt(cone_obs.x * cone_obs.x + cone_obs.y * cone_obs.y);
```

用途：

- 计算 hit 置信度。
- 放大远距离观测噪声。

## 8. 当前车辆二维旋转

```cpp
Eigen::Matrix2d R_current = historical_T_veh_to_map.block<2, 2>(0, 0);
```

用途：

- 将车体系观测噪声旋转到地图系。

## 9. updateMap 函数签名

```cpp
void updateMap(
    double gx,
    double gy,
    double cone_x,
    double cone_y,
    float r,
    float g,
    float b,
    int type,
    double dist_to_sensor,
    const Eigen::Matrix2d& vehicle_R
)
```

参数含义：

- `gx, gy`：锥桶全局坐标。
- `cone_x, cone_y`：锥桶局部传感器坐标。
- `r,g,b`：可视化颜色。
- `type`：颜色类型。
- `dist_to_sensor`：观测距离。
- `vehicle_R`：车辆在地图中的二维旋转。

## 10. 观测向量

```cpp
Eigen::Vector2d z(gx, gy);
```

这里 `z` 是当前帧对某个锥桶位置的观测值。

## 11. 基于距离的 hit 置信度

代码：

```cpp
double max_sensing_range = params_.max_lidar_range;
float confidence = 1.0f - static_cast<float>(dist_to_sensor / max_sensing_range) * 0.6f;
confidence = std::max(0.4f, confidence);
double dynamic_l_hit = params_.l_hit * confidence;
```

含义：

- 越近的锥桶，观测越可信。
- 距离越远，hit 加分越少。
- 置信度最低不低于 0.4。

如果 `dist_to_sensor=0`：

```text
confidence = 1.0
```

如果 `dist_to_sensor=max_lidar_range`：

```text
confidence = 0.4
```

## 12. 各向异性观测噪声

车体系下标准差：

```cpp
double sigma_long = params_.sigma_long;
double sigma_lat = params_.sigma_lat + params_.lat_factor * std::abs(current_gyro_z_);
```

含义：

- 纵向噪声固定较大。
- 横向噪声会随转弯角速度增大。

车体系协方差：

```cpp
Eigen::Matrix2d R_body;
R_body << sigma_long * sigma_long, 0.0,
          0.0, sigma_lat * sigma_lat;
```

距离惩罚：

```cpp
double ratio = dist_to_sensor / params_.distance_factor;
double overall_penalty = 1.0 + (ratio * ratio);
R_body *= overall_penalty;
```

也就是说观测越远，协方差越大。

## 13. 锁图后的动态噪声

如果：

```cpp
map_locked_ == true
```

代码增加速度和角速度惩罚：

```cpp
double speed_penalty = params_.speed_factor * current_speed_;
double yaw_rate_penalty = params_.speed_factor * std::abs(current_gyro_z_);
double dynamic_factor = 1.0 + speed_penalty + yaw_rate_penalty;
overall_penalty *= dynamic_factor;
```

设计意图：

- 锁图后通常进入冲刺圈，速度更快。
- 高速和大角速度下感知误差变大。
- 因此降低新观测在滤波中的权重。

注意：代码中 `yaw_rate_penalty` 使用了 `speed_factor`，而不是读取到的 `gyro_factor`。这可能是实现上的小问题。

## 14. 噪声旋转到地图系

```cpp
Eigen::Matrix2d R = vehicle_R * R_body * vehicle_R.transpose();
```

含义：

- 观测误差在车体系定义。
- 地图更新在地图系进行。
- 所以需要旋转协方差。

## 15. 数据关联第一层：颜色门控

遍历全局地图：

```cpp
for (auto &map_cone : cone_map_) {
    if (map_cone.type != type)
        continue;
    ...
}
```

只有颜色类型一致的锥桶才可能匹配。

优点：

- 避免左右边界颜色混淆。
- 降低错误关联概率。

风险：

- 如果前端颜色识别错，可能创建重复锥桶。

## 16. 数据关联第二层：欧氏距离粗筛

```cpp
Eigen::Vector2d delta = z - map_cone.pos;
if (delta.norm() > params_.max_match_dist)
    continue;
```

参数：

```text
max_match_dist
```

作用：

- 防止相距太远的点被马氏距离误关联。
- 降低计算量。

## 17. 数据关联第三层：马氏距离

协方差叠加：

```cpp
Eigen::Matrix2d S = map_cone.P + R;
```

距离：

```cpp
double m_dist = std::sqrt(delta.transpose() * S.inverse() * delta);
```

如果：

```cpp
m_dist < min_mahalanobis
```

更新最佳匹配：

```cpp
min_mahalanobis = m_dist;
best_match = &map_cone;
```

初始门限：

```cpp
double min_mahalanobis = params_.mahalanobis_thresh;
```

因此只有马氏距离小于阈值的候选才会被选中。

## 18. 匹配成功：卡尔曼更新

如果找到 `best_match`：

### 18.1 加过程噪声

```cpp
Eigen::Matrix2d Q_noise = Eigen::Matrix2d::Identity() * params_.kf_q_base;
best_match->P += Q_noise;
```

避免 P 趋近于 0 后过度自信。

### 18.2 计算卡尔曼增益

```cpp
Eigen::Matrix2d S = best_match->P + R;
Eigen::Matrix2d K = best_match->P * S.inverse();
```

### 18.3 更新位置

```cpp
best_match->pos = best_match->pos + K * (z - best_match->pos);
```

### 18.4 更新协方差

```cpp
best_match->P = (Eigen::Matrix2d::Identity() - K) * best_match->P;
```

### 18.5 更新状态

```cpp
best_match->matched_this_frame = true;
best_match->existence_score += dynamic_l_hit;
best_match->existence_score = std::min(best_match->existence_score, params_.l_max);
best_match->is_stable = (best_match->existence_score > params_.l_stable);
```

## 19. 匹配失败：新锥桶初始化

如果没有找到最佳匹配：

### 19.1 锁图后不新增

```cpp
if (map_locked_) return;
```

循迹模式第一圈锁图后，后续不允许地图增加新锥桶。

### 19.2 阿克曼 ROI 过滤

```cpp
if (!isInAckermannTube(cone_x, cone_y)) {
    return;
}
```

如果新锥桶不在车辆可能行驶路径附近，不加入地图。

### 19.3 创建新锥桶

```cpp
GlobalCone new_cone;
new_cone.id = next_cone_id_++;
new_cone.pos = z;
new_cone.P = Eigen::Matrix2d::Identity() * params_.kf_p_init;
new_cone.r = r;
new_cone.g = g;
new_cone.b = b;
new_cone.type = type;
new_cone.is_stable = false;
new_cone.matched_this_frame = true;
new_cone.existence_score = dynamic_l_hit;
cone_map_.push_back(new_cone);
```

新锥桶初始状态：

- 位置为当前观测。
- 初始协方差为 `kf_p_init * I`。
- 初始存在评分为距离动态加权 hit。
- 还不是稳定锥桶。

## 20. updateMap 复现伪代码

```cpp
void updateMap(observation z, local cone_xy, type) {
    confidence = 1 - dist/max_range * 0.6;
    confidence = max(0.4, confidence);
    hit = l_hit * confidence;

    R_body = diag(sigma_long^2, sigma_lat^2);
    R_body *= 1 + (dist / distance_factor)^2;

    if (map_locked) {
        R_body *= 1 + speed_factor * speed + speed_factor * abs(yaw_rate);
    }

    R = vehicle_R * R_body * vehicle_R.transpose();

    best = null;
    best_dist = mahalanobis_thresh;

    for cone in cone_map:
        if cone.type != type:
            continue;

        delta = z - cone.pos;
        if norm(delta) > max_match_dist:
            continue;

        S = cone.P + R;
        d = sqrt(delta.T * inverse(S) * delta);

        if d < best_dist:
            best_dist = d;
            best = cone;

    if best != null:
        best.P += Q;
        K = best.P * inverse(best.P + R);
        best.pos = best.pos + K * (z - best.pos);
        best.P = (I - K) * best.P;
        best.score += hit;
        best.score = min(best.score, l_max);
        best.stable = best.score > l_stable;
        best.matched_this_frame = true;
    else:
        if map_locked:
            return;
        if !in_ackermann_tube(cone_xy):
            return;
        create_new_cone(z, type, hit);
}
```

## 21. 核心价值总结

本工程的锥桶入图不是简单追加点，而是：

```text
颜色门控
  -> 欧氏粗筛
  -> 马氏距离数据关联
  -> 动态协方差建模
  -> 卡尔曼位置更新
  -> hit/miss 存在概率管理
  -> 稳定阈值发布
```

这是当前工程最值得汇报的核心技术链。
