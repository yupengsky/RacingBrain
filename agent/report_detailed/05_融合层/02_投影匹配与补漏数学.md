# 投影匹配与补漏数学

本章详细解释 `fs_fusion_box_math.cpp` 中三个核心函数：

```cpp
project_3d_boxes_to_2d
fuse_measurements
recover_missing_cones
```

## 1. CalibrationParams

定义在：

```text
include/fs_fusion_box/fs_fusion_box_math.hpp
```

结构：

```cpp
struct CalibrationParams {
    cv::Mat K;
    cv::Mat D;
    Eigen::Matrix4d T_l2c;
    int img_w;
    int img_h;
};
```

字段含义：

- `K`：相机内参矩阵。
- `D`：畸变系数。
- `T_l2c`：LiDAR 到 Camera 的 4x4 外参。
- `img_w`：图像宽度。
- `img_h`：图像高度。

内参矩阵：

```text
K = [ fx  0  cx
      0  fy  cy
      0   0   1 ]
```

## 2. ProjectedBox

定义：

```cpp
struct ProjectedBox {
    cv::Rect rect;
    int original_index;
    double depth;
    bool valid;
};
```

字段含义：

- `rect`：LiDAR 3D bbox 投影到图像后的 2D 包围框。
- `original_index`：对应原始 LiDAR detection 的索引。
- `depth`：锥桶中心在相机坐标系下的深度。
- `valid`：投影是否有效。

## 3. 3D 框投影到 2D

函数：

```cpp
std::vector<ProjectedBox> project_3d_boxes_to_2d(
    const vision_msgs::msg::Detection3DArray::ConstSharedPtr& lidar_msg,
    const CalibrationParams& params
)
```

### 3.1 取 3D bbox 中心和尺寸

```cpp
double cx = detection.bbox.center.position.x;
double cy = detection.bbox.center.position.y;
double cz = detection.bbox.center.position.z;
double dx = detection.bbox.size.x / 2.0;
double dy = detection.bbox.size.y / 2.0;
double dz = detection.bbox.size.z / 2.0;
```

### 3.2 生成 8 个角点

3D bbox 角点：

```text
(cx ± dx, cy ± dy, cz ± dz)
```

代码用 `signs[8][3]` 枚举所有组合。

### 3.3 LiDAR 坐标转 Camera 坐标

对每个角点：

```cpp
Eigen::Vector3d pt_c = (params.T_l2c * pt_l).head<3>();
```

如果：

```cpp
pt_c.z() <= 0.1
```

说明点在相机后方或太近，跳过。

### 3.4 透视投影

手写投影：

```cpp
double u = (pt_c.x() * fx / pt_c.z()) + cx;
double v = (pt_c.y() * fy / pt_c.z()) + cy;
```

注意：

- 这里没有使用 `params.D` 做畸变校正。
- 没有调用 `cv::projectPoints`。
- 只做最基础针孔模型。

### 3.5 生成图像包围框

投影点集合：

```cpp
std::vector<cv::Point2f> img_pts;
```

生成包围框：

```cpp
cv::Rect bounding_rect = cv::boundingRect(img_pts);
```

裁剪到图像范围：

```cpp
bounding_rect = bounding_rect & cv::Rect(0, 0, params.img_w, params.img_h);
```

如果面积大于 0：

```cpp
proj_box.valid = true;
```

否则：

```cpp
proj_box.valid = false;
```

## 4. IoU 计算

函数：

```cpp
double calculate_overlap(const cv::Rect& box1, const cv::Rect& box2)
```

实现：

```cpp
cv::Rect intersection = box1 & box2;
if (intersection.area() <= 0) return 0.0;

double area_inter = intersection.area();
double area_union = box1.area() + box2.area() - area_inter;
return area_inter / area_union;
```

这就是标准 IoU：

```text
IoU = intersection_area / union_area
```

## 5. 融合匹配

函数：

```cpp
FusionResult fuse_measurements(
    const std::vector<ProjectedBox>& proj_boxes,
    const vision_msgs::msg::Detection3DArray::ConstSharedPtr& lidar_msg,
    const vision_msgs::msg::Detection2DArray::ConstSharedPtr& camera_msg,
    double overlap_threshold = 0.3
)
```

返回：

```cpp
struct FusionResult {
    std::vector<drd25_msgs::msg::Cone> fused_cones;
    std::vector<int> unmatched_camera_indices;
};
```

### 5.1 初始化相机匹配标记

```cpp
std::vector<bool> is_camera_matched(camera_msg->detections.size(), false);
```

用于最后找出哪些 YOLO 框没有被 LiDAR 匹配。

### 5.2 遍历 LiDAR 投影框

对于每个投影框，先创建输出锥桶：

```cpp
drd25_msgs::msg::Cone cone;
cone.x = raw_det.bbox.center.position.x;
cone.y = raw_det.bbox.center.position.y;
cone.color = drd25_msgs::msg::Cone::UNKNOWN;
```

默认颜色是 `UNKNOWN`。

这体现了融合策略：

```text
LiDAR 位置保留，颜色由 Camera 尝试补充。
```

### 5.3 找 IoU 最大的 YOLO 框

遍历所有相机检测：

```cpp
cv::Rect cam_rect(
    bbox.center.position.x - bbox.size_x/2,
    bbox.center.position.y - bbox.size_y/2,
    bbox.size_x,
    bbox.size_y
);
double iou = calculate_overlap(pbox.rect, cam_rect);
```

如果 `iou > max_iou`，更新最佳匹配。

注意：代码里没有跳过已经匹配过的相机框。注释说明可以加一对一约束，但当前为了容错允许重复参与候选。

### 5.4 判断匹配成功

条件：

```cpp
best_match_idx != -1 && max_iou > overlap_threshold
```

默认阈值：

```text
0.3
```

匹配成功后：

```cpp
int class_id = std::stoi(camera_msg->detections[best_match_idx].results[0].hypothesis.class_id);
if (class_id >= 0 && class_id <= 4) {
    cone.color = static_cast<uint8_t>(class_id);
}
is_camera_matched[best_match_idx] = true;
```

也就是说，颜色直接来自 YOLO 输出。

### 5.5 LiDAR 有，Camera 没匹配上

如果投影无效、没有 YOLO 框或 IoU 不够：

```text
cone.color 保持 UNKNOWN
```

这类锥桶仍会进入 `fused_cones`，但后续 SLAM 会跳过 `UNKNOWN`：

```cpp
if (cone_obs.color == drd25_msgs::msg::Cone::UNKNOWN)
    continue;
```

也就是说当前 SLAM 不把 Unknown 锥桶加入地图。

## 6. 未匹配相机框记录

LiDAR 遍历结束后：

```cpp
for (size_t i = 0; i < camera_msg->detections.size(); ++i) {
    if (!is_camera_matched[i]) {
        result.unmatched_camera_indices.push_back(i);
    }
}
```

这些框会进入单目补漏。

## 7. 单目补漏

函数：

```cpp
std::vector<drd25_msgs::msg::Cone> recover_missing_cones(
    const std::vector<int>& unmatched_indices,
    const vision_msgs::msg::Detection2DArray::ConstSharedPtr& camera_msg,
    const CalibrationParams& params
)
```

目标：

```text
YOLO 看到了，但 LiDAR 没检测到的锥桶，估算一个 3D 位置。
```

### 7.1 Camera 到 LiDAR 变换

```cpp
Eigen::Matrix4d T_c2l = params.T_l2c.inverse();
```

### 7.2 单目距离估计

假设锥桶真实高度：

```cpp
double real_height = 0.30;
```

如果 bbox 高度太小：

```cpp
if (det.bbox.size_y < 5.0) continue;
```

用相似三角形估计深度：

```cpp
double z_c = (fy * real_height) / det.bbox.size_y;
```

横向坐标：

```cpp
double x_c = (det.bbox.center.position.x - cx) * z_c / fx;
```

代码固定：

```cpp
double y_c = 0.2;
```

然后：

```cpp
Eigen::Vector4d pt_cam(x_c, y_c, z_c, 1.0);
Eigen::Vector4d pt_lidar = T_c2l * pt_cam;
```

输出：

```cpp
cone.x = pt_lidar.x();
cone.y = pt_lidar.y();
cone.color = YOLO class_id;
```

## 8. 补漏精度评价

单目补漏的几何假设比较粗糙：

- 假设锥桶真实高度恒定 0.30 m。
- 使用 bbox 像素高度估计深度。
- `y_c` 直接设为 0.2。
- 没有地平面精确求交。
- 没有畸变校正。

所以补漏结果适合提升召回，不适合作为高可信位置来源。后续 SLAM 的存在评分和匹配更新可以对这些点进行进一步筛选。

## 9. 强制半径匹配与 IoU 匹配的关系

融合节点有两层匹配：

### 9.1 第一层：IoU 匹配

位置：

```cpp
fuse_measurements
```

依据：

```text
LiDAR 投影 2D 框与 YOLO 2D 框的重叠面积
```

适合外参准确、bbox 尺寸合理时。

### 9.2 第二层：中心距离强制匹配

位置：

```cpp
FusionNode::callback
```

依据：

```text
LiDAR 点投影中心与 YOLO 框中心像素距离
```

适合外参有轻微偏差、IoU 不稳定时。

风险：

```text
如果场景中锥桶密集，半径过大可能把颜色赋给错误锥桶。
```

## 10. 复现核心算法伪代码

```cpp
ProjectedBox project_box(Detection3D det, CalibrationParams p) {
    corners = generate_8_bbox_corners(det.center, det.size);
    image_points = [];

    for corner in corners:
        pc = p.T_l2c * corner;
        if pc.z <= 0.1:
            continue;
        u = p.fx * pc.x / pc.z + p.cx;
        v = p.fy * pc.y / pc.z + p.cy;
        image_points.push_back([u, v]);

    if image_points.empty:
        return invalid;

    rect = bounding_rect(image_points);
    rect = clip(rect, image_width, image_height);
    return ProjectedBox(rect, original_index, depth, valid=rect.area>0);
}

FusionResult fuse(projected, lidar, camera) {
    matched_camera = [false] * camera.size;
    result = [];

    for pbox in projected:
        cone.x = lidar[pbox.index].center.x;
        cone.y = lidar[pbox.index].center.y;
        cone.color = UNKNOWN;

        if pbox.valid:
            best_iou = 0;
            best = -1;
            for i, cam in enumerate(camera):
                rect = camera_bbox_to_rect(cam);
                iou = IoU(pbox.rect, rect);
                if iou > best_iou:
                    best_iou = iou;
                    best = i;

            if best != -1 and best_iou > 0.3:
                cone.color = camera[best].class_id;
                matched_camera[best] = true;

        result.fused_cones.push_back(cone);

    for i in camera indices:
        if !matched_camera[i]:
            result.unmatched_camera_indices.push_back(i);

    return result;
}
```

## 11. 实现注意点

1. 真实复现时建议把畸变校正补上。
2. `overlap_threshold=0.3` 是经验阈值，需要根据外参质量调参。
3. 如果 LiDAR bbox 太小或太大，IoU 会受影响。
4. 如果相机框检测到锥桶但 LiDAR 没有，补漏位置可能漂。
5. 如果 SLAM 不接受 UNKNOWN，融合层 Unknown 锥桶只在 debug marker 中有意义。
