# 一键运行与端到端验证

## 1. 最重要脚本

```text
scripts/run_dataset_slam_chain.sh
```

这个脚本是当前工程最完整的端到端验证入口。

它会同时验证：

- 相机输入。
- LiDAR 输入。
- GNSS/INS 输入。
- YOLO 输出。
- LiDAR 分割输出。
- 融合输出。
- SLAM 全局地图输出。

## 2. 一键演示命令

带 RViz：

```bash
cd /home/yupeng/GitHub/DRd26_SLAM
RVIZ=true KEEP_RUNNING=true ./scripts/run_dataset_slam_chain.sh
```

不带 RViz，只验证：

```bash
cd /home/yupeng/GitHub/DRd26_SLAM
./scripts/run_dataset_slam_chain.sh
```

## 3. 脚本参数

脚本支持通过环境变量控制：

```bash

BAG_RATE

MONITOR_TIMEOUT
```

```text
STARTUP_WAIT=12
MONITOR_TIMEOUT=150

RVIZ=false
```

## 4. 数据集路径

如果没有显式设置 `DATASET_DIR`，脚本读取：

```text
config/hardcoded_paths.ini
```

字段：

```ini
[datasets]
rosbag_2026_02_05 = /media/yupeng/新加卷/Datasets/rosbag2_2026_02_05-11_01_07
```

如果路径不存在，脚本退出。

## 5. 环境初始化

脚本执行：

```bash
source scripts/activate_ros_ml.sh
source install/setup.bash
```

这保证：

- ROS 2 可用。
- Python venv 可用。
- 自定义消息可被 Python monitor 导入。
- 本地依赖路径可用。

## 6. 日志目录

默认：

```text
log/runtime/dataset_slam_chain_<timestamp>
```

包含：

```text
perception.log
slam.log
bag.log
summary.json
```

## 7. 启动感知

脚本后台启动：

```bash
ros2 launch run_perception system_run.launch.py \
  >"${LOG_DIR}/perception.log" 2>&1 &
```

该 launch 会启动：

- `yolo_detector`
- `cone_segmentation_node`
- 延迟 2 秒启动 `fusion_box_node`

## 8. 启动 SLAM

脚本后台启动：

```bash
ros2 launch slam slam.launch.py "track:=${TRACK}" "rviz:=${RVIZ}" \
  >"${LOG_DIR}/slam.log" 2>&1 &
```

默认：

```text
track=acceleration
rviz=false
```

## 9. 回放 rosbag

等待节点启动：

```bash
sleep "${STARTUP_WAIT}"
```

然后播放：

```bash
ros2 bag play "${DATASET_DIR}" \
  --rate "${BAG_RATE}" \
  --topics /camera1/image_raw /lidar_points /gongji_gnss_ins_64
```

默认播放速率：

```text
0.25x
```

降低速率的原因：

- YOLO CPU 推理可能较慢。
- 点云处理也有计算量。
- 降低速率可以让链路更稳定。

## 11. 监控指标

monitor 统计：

```python
self.counts = {
    "/camera1/image_raw": 0,
    "/lidar_points": 0,
    "/gongji_gnss_ins_64": 0,
    "/yolo/cones": 0,
    "/cone_detection_custom": 0,
    "/perception/fusion/map": 0,
    "/global_map": 0,
}
```

还统计：

- 每个话题首次出现时间。
- 最大 YOLO 锥桶数。
- 最大 LiDAR 锥桶数。
- 最大融合锥桶数。
- 最大全局 marker 数。
- 非空全局地图消息数量。
- 最后融合消息时间戳。
- 最后全局地图时间戳。

## 12. 成功条件

脚本认为成功需要：

```python
node.counts["/perception/fusion/map"] > 0
and node.max_fused_cones > 0
and node.nonempty_global_messages > 0
```

同时要求该状态持续约 8 秒：

```python
if fused_ok and global_ok:
    if success_since is None:
        success_since = time.time()
    if time.time() - success_since >= 8.0:
        break
```

这可以避免偶发一帧误判成功。

## 14. 清理进程

脚本注册：

```bash
trap cleanup EXIT
```

退出时依次对：

```text
BAG_PID
PERCEPTION_PID
SLAM_PID
```

发送：

1. `SIGINT`
2. 等待 2 秒
3. `SIGTERM`

如果设置：

```bash
KEEP_RUNNING=true
```

成功后不会立即杀掉进程，而是保持运行，方便 RViz 查看。

## 17. 复现验收标准

复现成功的最低标准：

1. `/camera1/image_raw` 有图像输入。
2. `/lidar_points` 有点云输入。
3. `/gongji_gnss_ins_64` 有位姿输入。
4. `/yolo/cones` 有相机检测输出。
5. `/cone_detection_custom` 有 LiDAR 检测输出。
6. `/perception/fusion/map` 有非空融合锥桶。
7. `/global_map` 有非空 marker。

只要这 7 条成立，工程核心链路就已经跑通。
