# 核心功能复现路线图

## 1. 复现目标

如果要用 Python/C++ 从零复现本工程的核心功能，目标不是一开始就复现所有调试功能，而是先复现这条主链：

```text
Camera Image
  -> YOLO 2D cones

LiDAR PointCloud
  -> 3D cone clusters

YOLO 2D cones + LiDAR 3D cones
  -> fused drd25_msgs/Map

GNSS/INS + fused Map
  -> global cone map
```

这条链路就是工程的 100% 核心功能。

## 2. 第一步：定义消息

必须定义以下消息。

### 2.1 相机检测消息

```text
Cone.msg:
geometry_msgs/Point center
geometry_msgs/Vector3 size
float32 confidence
uint8 color

uint8 BLUE=0
uint8 RED=1
uint8 YELLOW_BIG=2
uint8 YELLOW_SMALL=3
uint8 UNKNOWN=4

ConeArray.msg:
std_msgs/Header header
Cone[] cones
```

### 2.2 LiDAR 检测消息

```text
ThreeDCone.msg:
geometry_msgs/Point center
geometry_msgs/Vector3 size

ThreeDConeArray.msg:
std_msgs/Header header
ThreeDCone[] cones
```

### 2.3 融合地图消息

```text
Cone.msg:
float64 x
float64 y
uint8 color

Map.msg:
std_msgs/Header header
Cone[] track
```

### 2.4 GNSS/INS 消息

至少要包含：

```text
std_msgs/Header header
float64 latitude
float64 longitude
float32 vel_e
float32 vel_n
float32 pitch
float32 roll
float32 yaw
float32 imu_gyro_z
```

## 3. 第二步：实现相机 YOLO 节点

语言：

```text
Python
```

依赖：

```text
rclpy
sensor_msgs
cv_bridge
ultralytics
cone_interfaces
```

逻辑：

1. 加载 YOLO 模型。
2. 订阅 `/camera1/image_raw`。
3. 图像转 OpenCV。
4. YOLO 推理。
5. 解析 bbox、类别、置信度。
6. 类别名映射颜色枚举。
7. 发布 `/yolo/cones`。
8. 可选发布 `/yolo/debug_image`。

最小输出必须保证：

```text
ConeArray.header = image_msg.header
```

## 4. 第三步：实现 LiDAR 分割节点

语言：

```text
C++
```

依赖：

```text
rclcpp
sensor_msgs
pcl_conversions
PCL
test_cone_segmentation messages
```

最小逻辑：

1. 订阅 `/lidar_points`。
2. `sensor_msgs/PointCloud2` 转 `pcl::PointCloud<pcl::PointXYZ>`。
3. 半径 ROI 截取。
4. VoxelGrid 降采样。
5. RANSAC 地面平面分割。
6. 提取非地面点。
7. EuclideanClusterExtraction 聚类。
8. AABB 尺寸筛选。
9. 发布 `ThreeDConeArray`。

可以先不实现 CSF 和圆锥拟合，只做 RANSAC + 几何筛选，就能复现主链路的大部分功能。

## 5. 第四步：实现融合节点

语言：

```text
C++
```

依赖：

```text
rclcpp
message_filters
vision_msgs
OpenCV
Eigen
drd25_msgs
cone_interfaces
test_cone_segmentation
```

逻辑：

1. 同步 `/yolo/cones` 和 `/cone_detection_custom`。
2. 将 3D cone 转为内部 3D bbox。
3. 将 2D cone 转为内部 2D bbox。
4. 加载相机内参和 LiDAR-Camera 外参。
5. 投影 3D bbox 的 8 个角点到图像。
6. 得到 LiDAR 投影 2D bbox。
7. 与 YOLO bbox 计算 IoU。
8. 匹配成功则输出 LiDAR x/y + YOLO color。
9. 未匹配 LiDAR 输出 Unknown。
10. 可选对未匹配 YOLO 做单目补漏。
11. 发布 `/perception/fusion/map`。

最小版本可以先不做单目补漏和强制半径匹配，只做 IoU。

## 6. 第五步：实现 SLAM 节点

语言：

```text
C++
```

依赖：

```text
rclcpp
message_filters
Eigen
PROJ
tf2_ros
nav_msgs
visualization_msgs
gnss_ins_msg
drd25_msgs
```

核心数据结构：

```cpp
struct GlobalCone {
    int id;
    Eigen::Vector2d pos;
    Eigen::Matrix2d P;
    float r, g, b;
    bool is_stable;
    double existence_score;
    bool matched_this_frame;
    int type;
};
```

节点逻辑：

1. 读取参数。
2. 初始化 PROJ。
3. 高频订阅 GNSS 发布 TF/Odom/Path。
4. 同步订阅 GNSS 和 `/perception/fusion/map`。
5. 第一帧 GNSS 设置局部地图原点。
6. 经纬度转 UTM，再转局部 map 坐标。
7. 姿态转车辆旋转矩阵。
8. 对每个融合锥桶：
   - 跳过 Unknown。
   - 局部坐标转全局。
   - 计算观测协方差。
   - 数据关联。
   - Kalman 更新或新建锥桶。
9. 帧末 miss 惩罚、删除低分锥桶、判定稳定。
10. 发布 `/global_map`。

## 7. 第六步：实现 launch

需要两个 launch：

### 7.1 perception launch

启动：

- YOLO 节点。
- LiDAR 分割节点。
- 融合节点。

并设置 remap：

```text
fusion input lidar  -> /cone_detection_custom
fusion input camera -> /yolo/cones
fusion output       -> /perception/fusion/map
```

### 7.2 slam launch

启动：

- `slam_node`
- 可选 RViz

加载：

```text
params.yaml
acceleration.yaml 或 autocross.yaml
```

## 8. 第七步：实现端到端验证脚本

脚本应做：

1. source ROS 环境。
2. source install 工作空间。
3. 启动感知。
4. 启动 SLAM。
5. 播放 rosbag。
6. 监听关键话题。
7. 检查融合地图和全局地图是否非空。
8. 输出 summary。

最小成功条件：

```text
/perception/fusion/map 非空
/global_map 非空
```

## 9. 功能复现优先级

### 必须复现

- 消息定义。
- YOLO 输出 `/yolo/cones`。
- LiDAR 输出 `/cone_detection_custom`。
- 融合输出 `/perception/fusion/map`。
- GNSS 经纬度转局部 map。
- 局部锥桶转全局。
- 颜色门控 + 欧氏距离 + 马氏距离。
- Kalman 更新。
- 存在评分。
- 稳定锥桶发布。

### 可以后复现

- HSV 检查。
- CSF 地面分割。
- 圆锥 RANSAC 拟合。
- 单目补漏。
- force match radius。
- 回环锁图。
- RViz mesh 美化。

### 当前工程未完成，不必强行复现

- skidpad 八字专用策略。
- 结构化全局地图数据发布。
- 图优化后端。
