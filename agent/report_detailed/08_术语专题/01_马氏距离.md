# 马氏距离

## 1. 马氏距离在本工程中的位置

源码位置：

```text
slam/slam/src/slam_node.cpp
```

函数：

```cpp
updateMap
```

核心代码：

```cpp
Eigen::Vector2d delta = z - map_cone.pos;
Eigen::Matrix2d S = map_cone.P + R;
double m_dist = std::sqrt(delta.transpose() * S.inverse() * delta);
```

用途：

```text
判断当前观测锥桶 z 是否应该匹配地图中已有锥桶 map_cone。
```

## 2. 为什么不用普通欧氏距离

欧氏距离：

```text
d = sqrt((x1-x2)^2 + (y1-y2)^2)
```

它只关心两个点相距多远，不关心：

- 当前观测是否噪声很大。
- 地图中已有锥桶是否不确定。
- 误差在纵向和横向是否不同。
- 车辆转弯时横向误差是否更大。

本工程的观测不确定度是动态变化的：

- 远距离观测更不可信。
- 高速冲刺更不可信。
- 转弯时横向误差更大。
- 地图中刚创建的锥桶不确定度更大。
- 多次观测融合后的锥桶不确定度更小。

因此需要马氏距离。

## 3. 马氏距离公式

对于观测 `z` 和地图锥桶位置 `x`：

```text
delta = z - x
S = P + R
d = sqrt(delta^T * S^-1 * delta)
```

其中：

- `z`：当前观测位置。
- `x`：地图中已有锥桶位置。
- `P`：地图中已有锥桶的位置协方差。
- `R`：当前观测的位置协方差。
- `S`：两种不确定度叠加后的总协方差。

## 4. 直观解释

如果地图和观测都很确定：

```text
P 小，R 小，S 小，S^-1 大
```

同样的 `delta` 会得到更大的马氏距离，说明系统不允许位置偏差太大。

如果地图或观测很不确定：

```text
P 大，或 R 大，S 大，S^-1 小
```

同样的 `delta` 会得到较小的马氏距离，说明系统允许更大的位置偏差。

这比欧氏距离更符合滤波系统。

## 5. 本工程的三层匹配门控

马氏距离不是唯一条件。`updateMap` 中有三层门控：

### 5.1 颜色门控

```cpp
if (map_cone.type != type)
    continue;
```

不同颜色不匹配。

### 5.2 欧氏距离粗筛

```cpp
if (delta.norm() > params_.max_match_dist)
    continue;
```

距离太远不匹配。

### 5.3 马氏距离精筛

```cpp
if (m_dist < min_mahalanobis) {
    min_mahalanobis = m_dist;
    best_match = &map_cone;
}
```

在候选中找马氏距离最小且小于阈值的锥桶。

## 6. 为什么还需要欧氏距离粗筛

理论上马氏距离已经考虑不确定度，似乎可以不用欧氏距离。

但工程中如果某个锥桶不确定度很大，`S` 很大，马氏距离可能会容忍很远的观测，导致错误关联。

所以代码先用：

```text
max_match_dist
```

硬性限制空间距离，再用马氏距离细分。

这是非常典型的工程防护。

## 7. 参数

YAML 中：

```yaml
mapping:
  mahalanobis_thresh: 2.5
  max_match_dist: 1.0
```

`mahalanobis_thresh` 越大：

- 更容易匹配已有锥桶。
- 重复点更少。
- 但错误关联风险更高。

`mahalanobis_thresh` 越小：

- 匹配更严格。
- 错误关联更少。
- 但重复锥桶更多。

`max_match_dist` 越大：

- 容忍位置漂移更大。
- 错误关联风险更高。

`max_match_dist` 越小：

- 关联更严格。
- 容易产生重影。

## 8. 最小实现

```cpp
GlobalCone* find_best_match(
    Eigen::Vector2d z,
    int type,
    Eigen::Matrix2d R_obs
) {
    double best_dist = mahalanobis_thresh;
    GlobalCone* best = nullptr;

    for (auto& cone : cone_map) {
        if (cone.type != type) continue;

        Eigen::Vector2d delta = z - cone.pos;
        if (delta.norm() > max_match_dist) continue;

        Eigen::Matrix2d S = cone.P + R_obs;
        double d = sqrt(delta.transpose() * S.inverse() * delta);

        if (d < best_dist) {
            best_dist = d;
            best = &cone;
        }
    }

    return best;
}
```

## 9. 汇报表达

可以这样讲：

```text
我们没有用简单欧氏距离匹配锥桶，而是使用马氏距离，把地图已有点的不确定度和当前观测的不确定度同时纳入关联判断。
当观测远、速度快或转弯时，观测协方差会增大，系统自然放宽关联范围；当锥桶已经多次观测收敛后，协方差变小，系统会更严格地拒绝离群观测。
```
