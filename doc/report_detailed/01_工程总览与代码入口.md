# 01 工程总览与代码入口

## 1. 工程到底在做什么

这个工程做的是赛道锥桶地图。

输入有三类：

1. GNSS/INS
2. 相机图像
3. LiDAR 点云

输出也有三类：

1. 车辆实时轨迹和里程计
2. 当前帧的局部锥桶观测
3. 全局锥桶地图

从代码实现看，这个工程的主目标很明确：

- 用相机识别锥桶颜色
- 用 LiDAR 给锥桶位置
- 用融合层把颜色和位置合到一起
- 用 GNSS/INS 给每一帧观测一个全局位姿
- 用滤波和数据关联把一帧帧局部观测积累成全局锥桶地图

## 2. 主链路总图

可以把整个系统看成五段。

### 2.1 传感器接入

| 传感器 | 输入 | 输出话题 | 核心文件 |
| --- | --- | --- | --- |
| GNSS/INS | 串口二进制流 | `gongji_gnss_ins_64` | `gnss/cpp_pubsub/src/publisher_member_function.cpp` |
| 相机 | `sensor_msgs/Image` | `/yolo/cones` | `perception/src/cone_ws/src/cone_detector/cone_detector/yolo_detector.py` |
| LiDAR | `sensor_msgs/PointCloud2` | `/cone_detection_custom` | `perception/src/cone_segmentation_test_3d/src/test_cone_segmentation/src/test_cone_segmentation.cpp` |

### 2.2 感知

| 模块 | 核心动作 | 输出 |
| --- | --- | --- |
| YOLO 节点 | 2D 检测 + 类别映射 + 调试图 | `ConeArray` |
| LiDAR 分割节点 | 地面分割 + 聚类 + 锥桶拟合 | `ThreeDConeArray` |

### 2.3 融合

| 模块 | 核心动作 | 输出 |
| --- | --- | --- |
| `fs_fusion_box` | 3D 框投影到图像，和 2D 框做 IoU 匹配，再补远距纯视觉锥桶 | `/perception/fusion/map` |

### 2.4 SLAM

| 模块 | 核心动作 | 输出 |
| --- | --- | --- |
| `slam_node` | GNSS 转全局位姿，锥桶数据关联，卡尔曼更新，存在分数维护，回环锁图 | `/global_map`、`vehicle_path`、`vehicle_odom` |

### 2.5 可视化

| 模块 | 输出 |
| --- | --- |
| 融合节点 | `fusion/markers` |
| LiDAR 分割节点 | `/cone_bboxes`、`/cone_markers` |
| SLAM 节点 | `/global_map` |

## 4. 工程主消息

你只要把下面 6 个消息吃透，这个工程就通了。

### 4.1 GNSS 高精度消息

文件：

- `gnss/gnss_ins_msg/msg/Gnssins64.msg`

核心字段：

- `latitude`
- `longitude`
- `height`
- `vel_e`
- `vel_n`
- `pitch`
- `roll`
- `yaw`
- `imu_gyro_z`
- `gnss_week`
- `gnss_second`

作用：

- 给 SLAM 提供绝对位置
- 给 SLAM 提供车体姿态
- 给 SLAM 提供速度和角速度
- 给整个系统提供统一时间戳

### 4.2 相机锥桶消息

文件：

- `perception/src/cone_ws/src/cone_interfaces/msg/Cone.msg`
- `perception/src/cone_ws/src/cone_interfaces/msg/ConeArray.msg`

核心字段：

- `center.x`
- `center.y`
- `size.x`
- `size.y`
- `confidence`
- `color`

这套消息表达的是图像里的 2D 框。

单位是像素。

### 4.3 LiDAR 锥桶消息

文件：

- `perception/src/cone_segmentation_test_3d/src/test_cone_segmentation/msg/ThreeDCone.msg`
- `perception/src/cone_segmentation_test_3d/src/test_cone_segmentation/msg/ThreeDConeArray.msg`

核心字段：

- `center`
- `size`

这套消息表达的是点云坐标系里的 3D 包围框。

### 4.4 融合输出消息

文件：

- `slam/drd25_msgs/msg/Cone.msg`
- `slam/drd25_msgs/msg/Map.msg`

核心字段：

- `Cone.x`
- `Cone.y`
- `Cone.color`
- `Map.track`

这里的 `x`、`y` 还是局部坐标。坐标系在 LiDAR 或车辆附近的局部系里。它还没进全局地图。

### 4.5 全局地图内部对象

文件：

- `slam/slam/include/slam/loop_closure_detector.hpp`

结构体：

```cpp
struct GlobalCone {
    int id;
    Eigen::Vector2d pos;
    Eigen::Matrix2d P;
    float r, g, b;
    bool is_stable = false;
    double existence_score = 0.0;
    bool matched_this_frame = false;
    int type;
};
```

这个结构体很关键。

它说明全局地图里每个锥桶至少要带下面这些状态：

1. 全局位置
2. 位置不确定度
3. 颜色
4. 是否稳定
5. 当前帧是否匹配
6. 生存分数

## 6. 启动顺序

工程推荐的实际启动顺序写在两个地方。

### 6.1 感知链启动

文件：

- `perception/src/run_perception/launch/system_run.launch.py`

它做了三件事：

1. 启动 YOLO 节点
2. 启动 LiDAR 分割节点
3. 延迟 2 秒再启动融合节点

### 6.2 SLAM 启动

文件：

- `slam/slam/launch/slam.launch.py`

它会：

1. 读取 `track` 参数
2. 加载 `params.yaml`
3. 再叠加 `acceleration.yaml`、`autocross.yaml` 或 `skidpad.yaml`
4. 启动 `slam_node`
5. 按需启动 RViz

### 6.3 一键链路

文件：

- `scripts/run_dataset_slam_chain.sh`

这个脚本先做环境准备，再起感知链、SLAM、rosbag 播放，最后用一个监控节点检查：

- `/perception/fusion/map` 有没有数据
- `/global_map` 有没有稳定锥桶

这个脚本很适合做端到端验收。

## 8. 这个工程的技术重心在哪里

从复现角度看，真正的技术重心有 4 个。

### 8.1 GNSS 位姿落全局

`slam_node.cpp` 里用 PROJ 把经纬度转 UTM，再减原点，得到全局平面坐标。

### 8.2 局部锥桶观测生成

融合节点把：

- LiDAR 的位置
- 相机的颜色

合成一帧局部锥桶列表。

### 8.3 数据关联

SLAM 用颜色、欧氏距离、马氏距离给当前观测找历史锥桶。

### 8.4 地图生存机制

地图不会“看到就收，收了就永远留”。

它会：

- 对匹配成功的锥桶加分
- 对该看见却没看见的锥桶减分
- 分数到稳定阈值才发布
- 分数低到下限就删除

这套机制很重要。它让地图能自己压噪声。

## 9. 现在就能下的判断

读完总览，你可以先下这几个判断。

### 9.1 这是一套 GNSS 约束下的锥桶地图系统

它不做激光 scan matching，也不做图优化后端。

它更接近：

`GNSS/INS 提供全局姿态 + 锥桶观测滤波建图`

### 9.2 这套代码很适合工程复现

原因很简单：

- 数据链路清楚
- 节点边界清楚
- 消息边界清楚
- 地图状态简单
- 核心代码集中

### 9.3 真正要抄住的核心文件很少

如果你只为了复现 80% 功能、100% 核心能力，重点盯住下面 6 个文件就够了：

1. `gnss/cpp_pubsub/src/publisher_member_function.cpp`

3. `perception/src/cone_segmentation_test_3d/src/test_cone_segmentation/src/test_cone_segmentation.cpp`
4. `perception/src/fs_fusion_box/src/fs_fusion_box_math.cpp`
