4. 3D 投影到 2D
5. IoU 配对
6. 得到 `FusedCone[]`

## 阶段 3：再做全局位姿

顺序建议：

1. 经纬度转 UTM
2. 第一帧设地图原点
3. yaw pitch roll 转旋转矩阵
4. 组 `T_vehicle_to_map`

这一步完成后，局部锥桶观测就能投到全局平面。

## 阶段 4：最后做地图更新

你只要实现下面 4 个函数，地图就出来了：

1. `transform_local_cone_to_global`
2. `find_best_match`
3. `kalman_update_or_insert`
4. `maintain_map_and_publish`

## 阶段 5：补赛道模式

最后再加：

1. FOV miss 惩罚
2. 阿克曼 ROI
3. 回环检测
4. 锁图逻辑

这时你的系统就会很接近原工程。

## 3. 最小核心实现

下面这份伪代码可以直接拿去写 Python 或 C++ 版本。

```python
class GlobalCone:
    def __init__(self, cone_id, x, y, color, p_init, score):
        self.id = cone_id
        self.pos = np.array([x, y], dtype=float)
        self.P = np.eye(2) * p_init
        self.color = color
        self.existence_score = score
        self.matched_this_frame = True
        self.is_stable = False


def build_pose_from_gnss(gnss, proj_transform, origin):
    utm_e, utm_n = proj_transform(gnss.latitude, gnss.longitude)
    if origin is None:
        origin = (utm_e, utm_n)
    tx = utm_e - origin[0]
    ty = utm_n - origin[1]
    R_vehicle = ypr_to_rotation(gnss.yaw, gnss.pitch, gnss.roll) @ R_fix()
    T = np.eye(4)
    T[:3, :3] = R_vehicle
    T[:3, 3] = [tx, ty, 0.0]
    return T, origin


def local_cone_to_global(cone, T_vehicle_to_map, T_lidar_to_vehicle):
    p_lidar = np.array([cone.local_x, cone.local_y, 0.0, 1.0])
    p_global = T_vehicle_to_map @ T_lidar_to_vehicle @ p_lidar
    return p_global[:2]


def build_obs_cov(dist, yaw_rate, speed, params, R_vehicle_2d, map_locked):
    sigma_long = params.sigma_long
    sigma_lat = params.sigma_lat + params.lat_factor * abs(yaw_rate)
    R_body = np.array([
        [sigma_long ** 2, 0.0],
        [0.0, sigma_lat ** 2],
    ])
    penalty = 1.0 + (dist / params.distance_factor) ** 2
    if map_locked:
        penalty *= 1.0 + params.speed_factor * speed + params.speed_factor * abs(yaw_rate)
    return R_vehicle_2d @ (R_body * penalty) @ R_vehicle_2d.T


def find_best_match(z, color, cone_map, R_obs, params):
    best = None
    best_m = params.mahalanobis_thresh
    for cone in cone_map:
        if cone.color != color:
            continue
        delta = z - cone.pos
        if np.linalg.norm(delta) > params.max_match_dist:
            continue
        S = cone.P + R_obs
        m = np.sqrt(delta.T @ np.linalg.inv(S) @ delta)
        if m < best_m:
            best_m = m
            best = cone
    return best


def update_map_with_observation(obs, z, cone_map, params, R_obs, dynamic_l_hit, map_locked, in_ackermann_tube):
    best = find_best_match(z, obs.color, cone_map, R_obs, params)
    if best is not None:
        Q = np.eye(2) * params.kf_q_base
        best.P = best.P + Q
        S = best.P + R_obs
        K = best.P @ np.linalg.inv(S)
        best.pos = best.pos + K @ (z - best.pos)
        best.P = (np.eye(2) - K) @ best.P
        best.matched_this_frame = True
        best.existence_score = min(best.existence_score + dynamic_l_hit, params.l_max)
        best.is_stable = best.existence_score > params.l_stable
        return

    if map_locked:
        return
    if not in_ackermann_tube(obs.local_x, obs.local_y):
        return

    cone_map.append(GlobalCone(next_id(), z[0], z[1], obs.color, params.kf_p_init, dynamic_l_hit))
```

这段代码已经覆盖了原工程最核心的地图状态转移。

## 4. 必须保留的关键公式

## 4.1 经纬度转平面坐标

原工程用的是 PROJ。

源码：

- `slam/slam/src/slam_node.cpp:48-56`
- `slam/slam/src/slam_node.cpp:244-247`

复现时你可以继续用 PROJ。

也可以用 `pyproj`。

核心点有两个：

1. 选对 UTM 区号
2. 第一帧设原点

公式上可以写成：

```text
[E, N] = UTM(latitude, longitude)
x_map = E - E_origin
y_map = N - N_origin
```

## 4.2 LiDAR 3D 目标投影到图像

源码：

- `perception/src/fs_fusion_box/src/fs_fusion_box_math.cpp:43-78`

公式：

```text
p_cam = T_lidar_to_camera * p_lidar
u = fx * Xc / Zc + cx
v = fy * Yc / Zc + cy
```

这个工程用 3D box 的 8 个角点投影，再取 2D 包围框。

## 4.3 IoU 融合

源码：

- `fs_fusion_box_math.cpp:86-169`

公式：

```text
IoU = area(intersection) / area(union)
```

然后取最大 IoU 的那个 2D 框。

## 4.4 远距纯视觉深度估计

源码：

- `fs_fusion_box_math.cpp:185-214`

公式：

```text
z_c = fy * real_height / bbox_height
x_c = (u - cx) * z_c / fx
```

代码里固定：

- `real_height = 0.30`
- `y_c = 0.2`

这两个值都很工程化。

### 你要完全复现原工程

就按这两个常数写。

### 你要做自己的版本

建议把：

- 锥桶真实高度
- 相机离地高度

改成配置参数。

## 4.5 视场判断

源码：

- `slam/slam/src/slam_node.cpp:604-626`

公式：

```text
p_local = T_current_pose^-1 * p_global
dist = sqrt(x^2 + y^2)
angle = atan2(|y|, x)
```

满足下面条件才算在视场里：

- `x > 0`
- `dist > blind_range`
- `dist < max_range`
- `angle < half_fov`

## 4.6 阿克曼管道 ROI

源码：

- `slam/slam/src/slam_node.cpp:628-648`

### 直线

```text
|y| < half_tube_width
0 < x < max_tube_length
```

### 转弯

```text
R = v / yaw_rate
lateral_deviation = | distance_to_turn_circle - |R| |
lateral_deviation < half_tube_width
```

## 4.7 马氏距离

源码：

- `slam/slam/src/slam_node.cpp:692-721`

公式：

```text
delta = z - x
S = P + R
d = sqrt(delta^T * S^-1 * delta)
```

这里的含义很直接：

- `z` 是当前观测
- `x` 是地图里旧锥桶位置
- `P` 是旧锥桶协方差
- `R` 是当前观测协方差

## 4.8 卡尔曼更新

源码：

- `slam/slam/src/slam_node.cpp:723-744`

公式：

```text
P_prior = P + Q
S = P_prior + R
K = P_prior * S^-1
x_new = x_old + K * (z - x_old)
P_new = (I - K) * P_prior
```

这里是最标准的二维线性卡尔曼更新。

## 4.9 存在分数

源码：

- `slam/slam/src/slam_node.cpp:656-662`
- `slam/slam/src/slam_node.cpp:739-742`
- `slam/slam/src/slam_node.cpp:399-407`
- `slam/slam/src/slam_node.cpp:457-467`

这就是一套加减分规则。

### 命中

```text
score += dynamic_l_hit
```

### 漏检

```text
score += l_miss
```

### 删除

```text
score < l_min  -> erase
```

### 稳定

```text
score > l_stable -> stable
```

这个工程没有把它直接写成概率滤波器。

但注释里把它和 log-odds 思想对应起来了。

## 5. 直接照抄时最容易漏掉的实现细节

这部分很重要。

很多人复现失败，原因很常见：细节漏了。

### 5.1 融合输出还在局部坐标

`/perception/fusion/map` 里的 `Cone.x/y` 还是局部坐标。

全局化发生在 `slam_node.cpp` 里。

### 5.2 感知时间戳必须继承原始消息

相机和 LiDAR 都这么做了。

如果你中间重新造时间戳，SLAM 的同步回调就会错位。

### 5.3 SLAM 用的是同步历史位姿

建图直接拿和感知消息对齐的那一帧 GNSS 位姿。

### 5.4 锁图后不再加新锥桶

这个行为会直接改变地图形态。

### 5.5 新锥桶要过阿克曼 ROI

如果你漏掉这一步，地图里会多很多杂点。

### 5.6 `GlobalCone` 只维护二维位置

地图没有维护锥桶高度，也没有维护 3D 姿态。

### 5.7 只发布稳定锥桶

如果你把所有候选都发出去，效果会很乱。

## 6. 当前代码里几处很值得记下来的事实

这些内容都来自源码本身。

做技术报告时可以直接讲。

### 6.1 `skidpad` 还没实现

代码：

- `slam/slam/src/slam_node.cpp:509-517`

### 6.2 LiDAR 分割配置文件和代码并不完全对齐

例如：

- 代码声明的是 `ground_ransac_dist_threshold`
- yaml 里写的是 `ground_distance_threshold`

代码位置：

- `test_cone_segmentation.cpp:37-39`
- `test_cone_segmentation.cpp:61-62`

配置位置：

- `cone_detection_params.yaml:8-10`

再比如：

- yaml 里的 `dbscan_eps`
- yaml 里的 `dbscan_min_points`

在当前 C++ 主流程里没有对应读取逻辑。

所以你复现时要以 C++ 源码为准。

### 6.3 `min_cluster_size` 被强行压到 3

代码：

- `test_cone_segmentation.cpp:73-75`

```cpp
cfg_.min_cluster_size = this->get_parameter("min_cluster_size").as_int();
if(cfg_.min_cluster_size > 3) cfg_.min_cluster_size = 3;
```

这说明 yaml 里就算填更大的值，运行时也会被压回 3。

### 6.4 `gyro_factor` 当前没有真正进角速度惩罚

代码：

- `slam_node.cpp:188-189`
- `slam_node.cpp:210-211`
- `slam_node.cpp:680-681`

参数被声明了，也被读取了。

但当前惩罚项这一行还是用了 `speed_factor`。

### 6.5 GNSS 节点的串口循环写在构造函数里

代码：

- `publisher_member_function.cpp:266-416`

这是一种能跑的写法，但不算常规。

### 6.6 工程里有两份 `drd25_msgs`

路径：

- `perception/src/drd25_msgs`
- `slam/drd25_msgs`

其中感知目录下那份带 `COLCON_IGNORE`。

文件：

- `perception/src/drd25_msgs/COLCON_IGNORE`

意思很清楚：

真正参与构建的是 `slam/drd25_msgs` 这一份。

## 9. 做汇报时可以怎么讲

建议你就按这 6 句话讲。

### 第一句

系统输入是 GNSS/INS、相机、LiDAR。

### 第二句

相机给锥桶颜色，LiDAR 给锥桶位置。

### 第三句

融合节点把两者变成一帧局部锥桶观测。

### 第四句

SLAM 节点用 GNSS/INS 把这帧观测投到全局坐标。

### 第五句

地图后端用颜色约束、马氏距离、卡尔曼更新和存在分数维护全局锥桶。

### 第六句

绕桩模式在第一圈回环成功后锁图，后续只在稳定地图上运行。

这 6 句话就能把工程主线讲清楚。
