## 你先记住这几个结论

### 1. 地图的基本单元是锥桶

这个工程最后维护的是锥桶对象。

它维护的是 `GlobalCone`。每个锥桶有：

- 全局二维坐标 `pos`
- 协方差 `P`
- 颜色和类型
- 存在分数 `existence_score`
- 是否稳定 `is_stable`

定义在：

- `slam/slam/include/slam/loop_closure_detector.hpp`

更新逻辑在：

- `slam/slam/src/slam_node.cpp`

### 2. GNSS 负责全局位姿

GNSS/INS 给车的绝对位置和姿态。

实时建图 没有做图优化，也没有做激光里程计。它直接用 GNSS/INS 的位姿，把局部锥桶观测投到全局坐标里。

所以这套系统更像：

`GNSS约束下的锥桶地图滤波器`

### 3. 融合节点只做局部观测整理

融合层不输出全局地图。

它做三件事：

1. 用外参把 LiDAR 3D 框投到图像。
2. 用 IoU 把 3D 框和 YOLO 2D 框配上。
3. 产出一帧局部坐标下的带颜色锥桶列表。

输出消息是：

- `/perception/fusion/map`
- 消息类型 `drd25_msgs/msg/Map`

### 4. 全局地图靠“匹配 + 卡尔曼更新 + 生存分数”长出来

核心函数是：

- `SlamProcessor::updateMap`

核心动作是：

1. 把当前锥桶观测变成全局坐标。
2. 用欧氏距离粗筛。
3. 用马氏距离找旧锥桶。
4. 匹配成功就做卡尔曼更新。
5. 没匹配上就新建锥桶。
6. 每帧再做一次 miss 惩罚和稳定性判定。
7. 只发布稳定锥桶。

## 一句话总结工程

这是一套基于 ROS2 的锥桶感知与地图系统。它用相机和 LiDAR 先做局部锥桶观测，再用 GNSS/INS 把观测投到全局坐标里，最后靠数据关联、卡尔曼更新、存在分数和回环锁图，得到一张稳定的全局锥桶地图。

# 01 工程总览与代码入口

## 1. 工程到底在做什么

这个工程做的是赛道锥桶地图。

输入有三类：

1. GNSS/INS
2. 相机图像
3. LiDAR 点云

输出也有三类：

1. 车辆实时轨迹和里程计
2. 当前帧的局部锥桶观测
3. 全局锥桶地图

从代码实现看，这个工程的主目标很明确：

- 用相机识别锥桶颜色
- 用 LiDAR 给锥桶位置
- 用融合层把颜色和位置合到一起
- 用 GNSS/INS 给每一帧观测一个全局位姿
- 用滤波和数据关联把一帧帧局部观测积累成全局锥桶地图

### 4.5 全局地图内部对象

文件：

- `slam/slam/include/slam/loop_closure_detector.hpp`

结构体：

```cpp
struct GlobalCone {
    int id;
    Eigen::Vector2d pos;
    Eigen::Matrix2d P;
    float r, g, b;
    bool is_stable = false;
    double existence_score = 0.0;
    bool matched_this_frame = false;
    int type;
};
```

这个结构体很关键。

它说明全局地图里每个锥桶至少要带下面这些状态：

1. 全局位置
2. 位置不确定度
3. 颜色
4. 是否稳定
5. 当前帧是否匹配
6. 生存分数

## 8. 这个工程的技术重心在哪里

从复现角度看，真正的技术重心有 4 个。

### 8.1 GNSS 位姿落全局

`slam_node.cpp` 里用 PROJ 把经纬度转 UTM，再减原点，得到全局平面坐标。

### 8.2 局部锥桶观测生成

融合节点把：

- LiDAR 的位置
- 相机的颜色

合成一帧局部锥桶列表。

### 8.3 数据关联

实时建图 用颜色、欧氏距离、马氏距离给当前观测找历史锥桶。

### 8.4 地图生存机制

地图不会“看到就收，收了就永远留”。

它会：

- 对匹配成功的锥桶加分
- 对该看见却没看见的锥桶减分
- 分数到稳定阈值才发布
- 分数低到下限就删除

这套机制很重要。它让地图能自己压噪声。

## 1. GNSS/INS 数据怎么进系统

核心文件：

- `gnss/cpp_pubsub/src/publisher_member_function.cpp`
- `gnss/gnss_ins_msg/msg/Gnssins64.msg`

### 1.1 节点启动方式

主函数在 `publisher_member_function.cpp:578-607`。

流程很直接：

1. 固定串口设备 `/dev/ttyUSB0`
2. 固定波特率 `460800`
3. `Stream::create_serial(...)`
4. 一直尝试连接
5. 连接成功后 `rclcpp::spin(std::make_shared<MinimalPublisher>())`

这里有一个很重要的实现细节。

`MinimalPublisher` 的构造函数里有一个 `while(1)` 串口读取循环，代码在 `publisher_member_function.cpp:266-416`。

主工作线程就卡在构造函数里，持续读串口，再直接发布消息。

这个写法很硬，但它能跑。

如果你自己重写，建议把串口读取放到：

- 独立线程
- 或定时器回调
- 或异步 IO

逻辑会更清楚。

### 1.2 数据帧格式怎么解

代码在 `publisher_member_function.cpp:202-399`。

关键点有 4 个：

1. 用 `findhead` 找帧头 `AA44AA45`
2. 单帧长度固定 `172` 字节
3. 用 `Calculate_Crc8` 验证前 171 字节
4. 按固定字节偏移解析字段

几个关键偏移：

- `+4` 纬度
- `+8` 经度
- `+12` 高程
- `+16` 东向速度
- `+18` 北向速度
- `+22` pitch
- `+24` roll
- `+26` yaw
- `+28` INS 状态
- `+29` GNSS week
- `+31` GNSS second

这些值都是原始整数，需要乘比例系数。

例子：

- 纬度、经度乘 `1e-7`
- 高度乘 `1e-3`
- 速度乘 `1e2 / 2^15`
- 姿态角乘 `180 / 2^15`

### 1.3 时间戳怎么来

代码在：

- `epoch2time`

这个方法不优雅，但很实用。

# 03 实时建图 建图与全局锥桶地图

## 1. 先给结论

这个工程做全局锥桶地图的办法很直接。

它没有做复杂图优化。

它的主方法是：

1. 用 GNSS/INS 给每一帧锥桶观测一个全局位姿。
2. 把局部锥桶坐标投到全局平面。
3. 对每个全局锥桶做数据关联。
4. 匹配上就卡尔曼更新。
5. 匹配不上就新建锥桶。
6. 每帧再做一次命中加分、漏检减分和稳定性判断。
7. 回环成功后锁图。

全局地图就这样一点点长出来。

## 7.4 第三步：构造观测噪声协方差

代码：

- `slam_node.cpp:664-690`

这部分是整套地图里最像“滤波算法”的地方。

### 先在车体系里建噪声

```cpp
sigma_long = params_.sigma_long
sigma_lat  = params_.sigma_lat + params_.lat_factor * abs(current_gyro_z_)
```

意思很清楚：

- 纵向噪声固定
- 横向噪声会随转弯变大

然后组成：

```cpp
R_body = [[sigma_long^2, 0],
          [0, sigma_lat^2]]
```

### 再按距离放大

代码：

- `ratio = dist_to_sensor / distance_factor`
- `overall_penalty = 1 + ratio^2`

距离越远，协方差越大。

### 锁图后再按车速和角速度放大

代码：

- `slam_node.cpp:679-685`

锁图后说明车已经进入更快的圈。

代码会再加一层惩罚。

这里有一个很值得记住的实现细节。

`yaw_rate_penalty` 这一行写的是：

```cpp
double yaw_rate_penalty = params_.speed_factor * std::abs(current_gyro_z_);
```

位置：

- `slam_node.cpp:681`

按变量命名看，这里本来很像想用 `gyro_factor`。

但当前代码实际复用了 `speed_factor`。

如果你要完全按原工程复现，这个行为要原样保留。

如果你要做自己的版本，建议把它改成：

```cpp
params_.gyro_factor * abs(current_gyro_z_)
```

### 最后转到地图系

代码：

- `slam_node.cpp:687-689`

```cpp
R = vehicle_R * R_body * vehicle_R.transpose()
```

这样观测协方差就跟车的朝向一起转到了地图坐标系。

## 7.5 第四步：做数据关联

代码：

- `slam_node.cpp:692-721`

数据关联分三层。

### 第一层：颜色必须一致

```cpp

```

### 第二层：欧氏距离粗筛

```cpp

```

这一步能挡掉很多离群点。

### 第三层：马氏距离精筛

```cpp
S = map_cone.P + R

```

从所有候选里取最小的那个。

## 7.6 第五步：匹配成功就做卡尔曼更新

代码：

- `slam_node.cpp:723-744`

流程：

1. 先给旧锥桶协方差加过程噪声 `Q`
2. 算卡尔曼增益 `K`
3. 更新位置
4. 更新协方差
5. 标记 `matched_this_frame = true`
6. 增加存在分数
7. 再判断是否稳定

具体公式：

```cpp
P = P + Q
S = P + R
K = P * S.inverse()
pos = pos + K * (z - pos)
P = (I - K) * P
```

命中奖励：

```cpp
existence_score += dynamic_l_hit
existence_score = min(existence_score, l_max)
```

## 13. 全局锥桶地图是怎么一步步长出来的

现在把整个过程压成一条链。

### 第 1 步

GNSS/INS 给出当前时刻车辆全局位姿。

### 第 2 步

融合节点给出当前时刻局部锥桶列表。

每个锥桶有：

- 局部 `x y`
- 颜色 `color`

### 第 3 步

实时建图 把局部锥桶通过：

```cpp
T_vehicle_to_map * T_lidar_to_vehicle * p_lidar
```

转成全局平面点。

### 第 4 步

对每个全局点，在 `cone_map_` 里找同色旧锥桶。

### 第 5 步

先用欧氏距离粗筛，再用马氏距离精筛。

### 第 6 步

匹配成功：

- 更新位置
- 更新协方差
- 增加存在分数

### 第 7 步

匹配失败：

- 看是否允许加新锥桶
- 看是否落在阿克曼 ROI
- 满足就新建

### 第 8 步

这一帧结束时，再对视场内没命中的旧锥桶减分。

### 第 9 步

分数超过稳定阈值，就标记稳定。

### 第 10 步

只把稳定锥桶发到 `/global_map`。

这 10 步叠起来，就是“全局锥桶地图”。

on_sync_callback(gnss_msg, fused_map_msg):
    T = build_vehicle_pose_from_gnss(gnss_msg)
    speed = calc_speed(gnss_msg)
    yaw_rate = calc_yaw_rate(gnss_msg)

    for cone in cone_map_:

    for obs in fused_map_msg.track:

        p_lidar = [obs.x, obs.y, 0, 1]
        p_global = T * T_l2v * p_lidar

        best = find_best_match_by_color_and_mahalanobis(p_global.xy, obs.color, R_obs)

        if best exists:
            kalman_update(best, p_global.xy, R_obs)
            best.existence_score += dynamic_hit(obs.distance)
            best.matched_this_frame = true
        else:
            if map_locked:
                continue
            if !in_ackermann_tube(obs.x, obs.y):
                continue
            add_new_cone(p_global.xy, obs.color)

    for cone in cone_map_:
        if in_fov(cone, T) and !cone.matched_this_frame:
            cone.existence_score += l_miss
        if cone.existence_score < l_min:
            erase cone
        cone.is_stable = cone.existence_score > l_stable

    if track_type == autocross and detect_loop():
        purge_unstable_cones()
        map_locked = true

    publish_only_stable_cones()

## 3. 最小核心实现

```python
class GlobalCone:
    def __init__(self, cone_id, x, y, color, p_init, score):
        self.id = cone_id
        self.pos = np.array([x, y], dtype=float)
        self.P = np.eye(2) * p_init
        self.color = color
        self.existence_score = score
        self.matched_this_frame = True
        self.is_stable = False


def build_pose_from_gnss(gnss, proj_transform, origin):
    utm_e, utm_n = proj_transform(gnss.latitude, gnss.longitude)
    if origin is None:
        origin = (utm_e, utm_n)
    tx = utm_e - origin[0]
    ty = utm_n - origin[1]
    R_vehicle = ypr_to_rotation(gnss.yaw, gnss.pitch, gnss.roll) @ R_fix()
    T = np.eye(4)
    T[:3, :3] = R_vehicle
    T[:3, 3] = [tx, ty, 0.0]
    return T, origin


def local_cone_to_global(cone, T_vehicle_to_map, T_lidar_to_vehicle):
    p_lidar = np.array([cone.local_x, cone.local_y, 0.0, 1.0])
    p_global = T_vehicle_to_map @ T_lidar_to_vehicle @ p_lidar
    return p_global[:2]


def build_obs_cov(dist, yaw_rate, speed, params, R_vehicle_2d, map_locked):
    sigma_long = params.sigma_long
    sigma_lat = params.sigma_lat + params.lat_factor * abs(yaw_rate)
    R_body = np.array([
        [sigma_long ** 2, 0.0],
        [0.0, sigma_lat ** 2],
    ])
    penalty = 1.0 + (dist / params.distance_factor) ** 2
    if map_locked:
        penalty *= 1.0 + params.speed_factor * speed + params.speed_factor * abs(yaw_rate)
    return R_vehicle_2d @ (R_body * penalty) @ R_vehicle_2d.T


def find_best_match(z, color, cone_map, R_obs, params):
    best = None
    best_m = params.mahalanobis_thresh
    for cone in cone_map:
        if cone.color != color:
            continue
        delta = z - cone.pos
        if np.linalg.norm(delta) > params.max_match_dist:
            continue
        S = cone.P + R_obs
        m = np.sqrt(delta.T @ np.linalg.inv(S) @ delta)
        if m < best_m:
            best_m = m
            best = cone
    return best


def update_map_with_observation(obs, z, cone_map, params, R_obs, dynamic_l_hit, map_locked, in_ackermann_tube):
    best = find_best_match(z, obs.color, cone_map, R_obs, params)
    if best is not None:
        Q = np.eye(2) * params.kf_q_base
        best.P = best.P + Q
        S = best.P + R_obs
        K = best.P @ np.linalg.inv(S)
        best.pos = best.pos + K @ (z - best.pos)
        best.P = (np.eye(2) - K) @ best.P
        best.matched_this_frame = True
        best.existence_score = min(best.existence_score + dynamic_l_hit, params.l_max)
        best.is_stable = best.existence_score > params.l_stable
        return

    if map_locked:
        return
    if not in_ackermann_tube(obs.local_x, obs.local_y):
        return

    cone_map.append(GlobalCone(next_id(), z[0], z[1], obs.color, params.kf_p_init, dynamic_l_hit))
```

这段代码已经覆盖了原工程最核心的地图状态转移。

## 4. 必须保留的关键公式

## 4.1 经纬度转平面坐标

原工程用的是 PROJ。

源码：

- `slam/slam/src/slam_node.cpp:48-56`
- `slam/slam/src/slam_node.cpp:244-247`

复现时你可以继续用 PROJ。

也可以用 `pyproj`。

核心点有两个：

1. 选对 UTM 区号
2. 第一帧设原点

公式上可以写成：

```text
[E, N] = UTM(latitude, longitude)
x_map = E - E_origin
y_map = N - N_origin
```

## 4.2 LiDAR 3D 目标投影到图像

源码：

- `perception/src/fs_fusion_box/src/fs_fusion_box_math.cpp:43-78`

公式：

```text
p_cam = T_lidar_to_camera * p_lidar
u = fx * Xc / Zc + cx
v = fy * Yc / Zc + cy
```

这个工程用 3D box 的 8 个角点投影，再取 2D 包围框。

## 4.3 IoU 融合

源码：

- `fs_fusion_box_math.cpp:86-169`

公式：

```text
IoU = area(intersection) / area(union)
```

然后取最大 IoU 的那个 2D 框。

## 4.4 远距纯视觉深度估计

源码：

- `fs_fusion_box_math.cpp:185-214`

公式：

```text
z_c = fy * real_height / bbox_height
x_c = (u - cx) * z_c / fx
```

代码里固定：

- `real_height = 0.30`
- `y_c = 0.2`

这两个值都很工程化。

### 你要完全复现原工程

就按这两个常数写。

### 你要做自己的版本

建议把：

- 锥桶真实高度
- 相机离地高度

改成配置参数。

## 4.5 视场判断

源码：

- `slam/slam/src/slam_node.cpp:604-626`

公式：

```text
p_local = T_current_pose^-1 * p_global
dist = sqrt(x^2 + y^2)
angle = atan2(|y|, x)
```

满足下面条件才算在视场里：

- `x > 0`
- `dist > blind_range`
- `dist < max_range`
- `angle < half_fov`

## 4.6 阿克曼管道 ROI

源码：

- `slam/slam/src/slam_node.cpp:628-648`

### 直线

```text
|y| < half_tube_width
0 < x < max_tube_length
```

### 转弯

```text
R = v / yaw_rate
lateral_deviation = | distance_to_turn_circle - |R| |
lateral_deviation < half_tube_width
```

## 4.7 马氏距离

源码：

- `slam/slam/src/slam_node.cpp:692-721`

公式：

```text
delta = z - x
S = P + R
d = sqrt(delta^T * S^-1 * delta)
```

这里的含义很直接：

- `z` 是当前观测
- `x` 是地图里旧锥桶位置
- `P` 是旧锥桶协方差
- `R` 是当前观测协方差

## 4.8 卡尔曼更新

源码：

- `slam/slam/src/slam_node.cpp:723-744`

公式：

```text
P_prior = P + Q
S = P_prior + R
K = P_prior * S^-1
x_new = x_old + K * (z - x_old)
P_new = (I - K) * P_prior
```

这里是最标准的二维线性卡尔曼更新。

## 4.9 存在分数

源码：

- `slam/slam/src/slam_node.cpp:656-662`
- `slam/slam/src/slam_node.cpp:739-742`
- `slam/slam/src/slam_node.cpp:399-407`
- `slam/slam/src/slam_node.cpp:457-467`

这就是一套加减分规则。

### 命中

```text
score += dynamic_l_hit
```

### 漏检

```text
score += l_miss
```

### 删除

```text
score < l_min  -> erase
```

### 稳定

```text
score > l_stable -> stable
```

这个工程没有把它直接写成概率滤波器。

但注释里把它和 log-odds 思想对应起来了。

## 5. 直接照抄时最容易漏掉的实现细节

这部分很重要。

很多人复现失败，原因很常见：细节漏了。

### 5.1 融合输出还在局部坐标

`/perception/fusion/map` 里的 `Cone.x/y` 还是局部坐标。

全局化发生在 `slam_node.cpp` 里。

### 5.2 感知时间戳必须继承原始消息

相机和 LiDAR 都这么做了。

如果你中间重新造时间戳，实时建图 的同步回调就会错位。

### 5.3 实时建图 用的是同步历史位姿

建图直接拿和感知消息对齐的那一帧 GNSS 位姿。

### 5.4 锁图后不再加新锥桶

这个行为会直接改变地图形态。

### 5.5 新锥桶要过阿克曼 ROI

如果你漏掉这一步，地图里会多很多杂点。

### 5.6 `GlobalCone` 只维护二维位置

地图没有维护锥桶高度，也没有维护 3D 姿态。

### 5.7 只发布稳定锥桶

如果你把所有候选都发出去，效果会很乱。
